JavaBeansTM Component API

 Documentation Contents
The JavaBeans component API extends the Java platform's Write Once, Run AnywhereTM capability to reusable component development. In fact, the JavaBeans architecture takes interoperability a major step forward -- your code runs on every OS and also within any application environment. A developer who creates components based on the JavaBeans architecture (beans) secures a future in the emerging network software market without losing customers that use proprietary platforms, because beans interoperate with ActiveX, OpenDoc, and LiveConnect.

As of v 1.4, the java.beans package provides support for long-term persistence -- reading and writing a bean as a textual representation of its property values. Other changes in v 1.4 include improvements to the Introspector class, a JavaBeans Specification amendment to support retrieving listeners from a bean, and miscellaneous improvements in the treatment of event listeners.
